Architecture

A Federated, Platform-Agnostic Design

NSDF connects existing institutional systems rather than replacing them, allowing data providers to retain control while participating in shared discovery, access, streaming, and analysis workflows.

1

Data and Instruments

Scientific facilities, repositories, observational systems, and simulations produce and maintain data.

2

Entry Points

A domain-agnostic software stack provides a uniform interface across small, medium, and large computing centers and cloud providers.

3

Federated Services

Catalogs, metadata, data movement, streaming, access control, provenance, dashboards, and workflow services connect the ecosystem.

4

AI and Computing

Researchers connect data to cloud, HPC, visualization, analytics, and AI-guided scientific workflows.

Networking

High-Speed National Connectivity

The NSDF pilot uses high-speed research networking, including Internet2 infrastructure, to support data movement among storage resources, computing sites, facilities, and entry points.

100 Gbps+Backbone-class connectivity supporting distributed scientific data movement

Deployment

NSDF Entry Points

Entry Points provide a uniform, domain-agnostic set of services that can be deployed at academic sites, computing centers, facilities, and cloud environments. Configurations can scale with the services and resources required by each site.
